# Foodia - 오늘 뭐먹지 근처맛집 서비스 설계안

## 📋 프로젝트 개요

### 서비스명
**Foodia** - 오늘 뭐먹지 근처맛집

### 서비스 목적
사용자가 지금 있는 위치 근처에서 빠르게 식당을 찾고, 고민 시간을 줄이도록 돕는 모바일 우선 반응형 웹 서비스

### 주요 타겟
- **지역:** 성수역 인근 맛집 중심
- **사용자:** 빠른 맛집 결정이 필요한 직장인, 대학생, 일반인
- **시나리오:** 점심/저녁 시간에 근처 식당을 빠르게 찾아야 하는 상황

### 개발 범위
- **팀 구성:** 4명의 개발자
- **개발 기간:** 5일 (MVP)
- **플랫폼:** 반응형 웹 (모바일 우선)

---

## 🎯 핵심 사용자 문제 및 해결 방식

### 문제 정의
사용자는 점심/저녁 시간에 근처 식당을 찾을 때:
1. 정보가 너무 많아서 결정하기 어렵다
2. 광고성 정보가 섞여 있어 신뢰하기 어렵다
3. 빠르게 결정할 수 있는 도구가 부족하다
4. 혼자 먹기 애매한 메뉴나 식당이 있다

### 해결 방식
1. **빠른 탐색:** 위치 기반 + 필터링으로 후보를 빠르게 좁힘
2. **직관적 비교:** 카드 UI로 핵심 정보만 한눈에 표시
3. **저장 기능:** 나중에 방문할 맛집을 저장해서 재방문
4. **같이먹기 게시판:** 익명으로 간단하게 식사 메이트 모집

## 📱 화면 목록 및 설계

### 1. 홈 / 온보딩 화면 (/)

**목적:** 사용자가 즉시 맛집 탐색을 시작할 수 있도록 유도

**주요 구성 요소:**
- 브랜드 헤더 (Foodia + 서비스 한줄 설명)
- 지역명 검색 입력창
- "내 주변 맛집 찾기" 메인 CTA 버튼
- 자주 찾는 지역 빠른 선택 (성수역, 서울숲, 뚝섬역)
- 음식 종류별 카테고리 버튼 (한식, 중식, 일식, 양식, 카페, 전체)
- 서비스 사용 팁 안내

**디자인 특징:**
- 그라데이션 배경 (오렌지 50 → 화이트)
- 큰 CTA 버튼으로 시선 집중
- 깔끔한 카드 기반 레이아웃
- 하단 탭바 네비게이션

---

### 2. 검색 결과 화면 (/search)

**목적:** 사용자가 필터를 통해 원하는 맛집을 빠르게 찾도록 지원

**주요 구성 요소:**
- 상단 검색바 (다른 지역 재검색 가능)
- 지역명 + 결과 개수 표시
- 필터 섹션
  - 음식 종류: 전체, 한식, 중식, 일식, 양식, 카페
  - 가격대: 전체, 1만원 이하, 1-2만원, 2만원 이상
  - 상황: 전체, 혼밥, 데이트, 친구모임
- 정렬 옵션: 거리순, 평점순
- 맛집 카드 리스트

**맛집 카드 구성:**
- 썸네일 이미지
- 식당 이름 + 카테고리
- 평점 + 리뷰 수
- 거리
- 태그 2개
- 가격대
- 북마크 버튼

**디자인 특징:**
- 스티키 헤더로 항상 검색 가능
- 수평 스크롤 필터 칩
- 그림자가 있는 카드 UI
- 빈 상태 UI (결과 없을 때)

---

### 3. 맛집 상세 화면 (/restaurant/:id)

**목적:** 맛집의 상세 정보를 제공하고, 다음 행동(저장/리뷰/같이먹기)으로 유도

**주요 구성 요소:**
- 대표 이미지 (전체 너비)
- 기본 정보
  - 식당 이름 + 카테고리 배지
  - 평점 + 리뷰 수
  - 주소
  - 영업시간
- 태그 섹션 (분위기, 가격대 등)
- 대표 메뉴 리스트 (메뉴명 + 가격)
- 같이먹기 모집 미리보기 (최대 2개)
- 리뷰 섹션
  - 리뷰 목록 (작성자, 별점, 내용, 태그, 날짜)
  - "리뷰 작성" CTA
- 하단 고정 액션바
  - "같이먹기" 버튼
  - "리뷰 작성" 버튼 (오렌지)

**디자인 특징:**
- 이미지 위에 반투명 뒤로가기/북마크 버튼
- 섹션별 구분선으로 명확한 계층 구조
- 하단 고정 액션바로 주요 행동 유도
- 같이먹기 섹션을 오렌지 배경으로 강조

---

### 4. 리뷰 작성 화면 (/review/write/:id)

**목적:** 사용자가 30초 내에 빠르게 리뷰를 작성할 수 있도록 간소화

**주요 구성 요소:**
- 식당 정보 요약 카드
- 별점 선택 (1-5점, 큰 아이콘)
- 한줄 리뷰 입력 (최대 500자)
- 태그 선택 (가성비, 분위기좋음, 조용함, 양많음, 맛있음, 친절함, 깔끔함, 재방문의사)
- "리뷰 등록하기" CTA

**디자인 특징:**
- 각 섹션을 카드로 구분
- 인터랙티브 별점 선택 (호버/클릭 효과)
- 선택 가능한 태그 칩
- 글자 수 카운터 표시

---

### 5. 저장 목록 화면 (/saved)

**목적:** 저장한 맛집을 다시 쉽게 찾아볼 수 있도록 제공

**주요 구성 요소:**
- 헤더 (저장한 맛집 + 설명)
- 필터 칩 (전체, 혼밥, 데이트, 친구모임)
- 저장 개수 표시
- 맛집 카드 리스트
- 빈 상태 UI (저장한 맛집 없을 때)

**디자인 특징:**
- 검색 결과와 동일한 카드 UI 재사용
- 북마크 아이콘 강조 (채워진 상태)
- 저장 취소 시 즉시 리스트에서 제거

---

### 6. 같이먹기 게시판 화면 (/together/:restaurantId)

**목적:** 해당 식당에서 같이 먹을 사람을 찾는 게시판

**주요 구성 요소:**
- 식당 정보 카드
- 안내 배너 (같이먹기 사용법)
- 모집글 개수 표시
- 모집글 카드 리스트
  - 익명 닉네임 + 작성일
  - 시간대 태그 (지금, 오늘 점심, 오늘 저녁 등)
  - 모집 내용
  - 상황 태그 + 모집 인원
  - 오픈채팅 링크 (있는 경우)
  - "관심있어요" 버튼 + 관심 수
- 플로팅 "글 작성하기" 버튼 (우하단)
- 빈 상태 UI (모집글 없을 때)

**디자인 특징:**
- 모집글 카드는 오렌지 배경으로 강조
- 시간대 태그를 우상단 배지로 표시
- 관심 표시 시 하트 아이콘 채워짐
- 플로팅 버튼은 원형 + 그림자

---

### 7. 같이먹기 글 작성 화면 (/together/write/:restaurantId)

**목적:** 빠르고 간단하게 같이먹기 모집글 작성

**주요 구성 요소:**
- 식당 정보 요약 카드
- 모집글 입력 (최대 200자)
- 희망 시간대 선택 (지금, 오늘 점심, 오늘 저녁, 내일 점심, 내일 저녁)
- 상황 태그 선택 (혼밥 탈출, 점심메이트, 저녁메이트, 카페, 친구모임)
- 모집 인원 선택 (± 버튼으로 조절, 1-10명)
- 오픈채팅 링크 입력 (선택사항)
- "모집글 등록하기" CTA

**디자인 특징:**
- 각 입력 항목을 카드로 구분
- 선택형 UI 중심 (칩, 버튼)
- 텍스트 입력은 최소화
- 글자 수 카운터 표시

---

### 8. 마이페이지 (/profile)

**목적:** 사용자의 활동 내역 요약 제공

**주요 구성 요소:**
- 프로필 카드
  - 익명 사용자 아이콘
  - 활동 통계 (저장한 맛집, 작성한 리뷰, 같이먹기)
- 메뉴 리스트
  - 저장한 맛집
  - 내 리뷰
  - 내 같이먹기 글
  - 설정
- 서비스 안내 배너

**디자인 특징:**
- 통계는 3열 그리드로 표시
- 메뉴는 카드 형태의 버튼
- 우측 화살표로 클릭 가능 표시
